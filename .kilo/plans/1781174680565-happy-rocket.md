# Plan: Fix Notification Persistence on Logout/Login

## Problem

When a user logs out and logs in with a different user, they see notification popups that were already dismissed. This happens because:

1. Notifications are stored in localStorage with key `notifications`
2. When the app loads from localStorage, `loadStateFromStorage()` sets `alreadyShownToast = false` for ALL notifications
3. This causes old notifications to be shown as toasts again when the user logs back in

The issue is that notifications are persisted in localStorage without being associated with a specific user. When a different user logs in, they see the previous user's notifications.

## Solution

We need to:
1. **Associate notifications with users** - Store notifications in localStorage with a user-specific key
2. **Clear notifications on logout** - When a user logs out, clear their notifications from localStorage
3. **Mark notifications as read when dismissed** - When a toast is dismissed, mark the notification as read
4. **Clear old notifications on login** - When a user logs in, clear any old notifications that don't belong to the current user

## Implementation

### 1. Associate notifications with users

In `src/contexts/NotificationContext.tsx`, we need to:
- Modify `STORAGE_KEY` to include a user identifier
- Get the user identifier from the session context or auth service
- Use the user-specific key for localStorage

### 2. Clear notifications on logout

In `src/services/authService.ts`, we need to:
- Clear the notifications from localStorage when the user logs out
- We can use the `logout` method to clear notifications

### 3. Mark notifications as read when dismissed

We already implemented this in the previous change. When a toast is dismissed, we call `removeNotification` which removes the notification from the panel.

### 4. Clear old notifications on login

In `src/contexts/NotificationContext.tsx`, we need to:
- When loading notifications from localStorage, check if they belong to the current user
- If not, clear them and start fresh

### 2. Modify `loadStateFromStorage` to check user

In `src/contexts/NotificationContext.tsx`, modify `loadStateFromStorage` to:
- Accept a user identifier parameter
- Check if the stored state belongs to the current user
- If not, return null (or empty state)

```typescript
function loadStateFromStorage(userId: string): Partial<NotificationState> | null {
  try {
    const savedState = localStorage.getItem(STORAGE_KEY);
    const state = JSON.parse(savedState || '{}') as Partial<NotificationState>;
    if (savedState && state.userId === userId) {
      for (const notification of Object.values(state.notifications || {}).flat()) {
        notification.alreadyShownToast = true; // Don't show toasts again
      }
      return state;
    }
    return null;
  } catch (error) {
    console.error('Failed to load notifications from localStorage:', error);
  }
  return null;
}
```

### 3. Update `saveStateToStorage` to include user ID

In `src/contexts/NotificationContext.tsx`, modify `saveStateToStorage` to:
- Accept a user identifier parameter
- Include the user ID in the saved state

```typescript
function saveStateToStorage(state: NotificationState, userId: string): void {
  try {
    const { openPanels, activeModal, ...stateToSave } = state;
    stateToSave.userId = userId;
    localStorage.setItem(STORAGE_KEY, JSON.stringify(stateToSave));
  } catch (error) {
    console.error('Failed to save notifications to localStorage:', error);
  }
}
```

### 5. Clear notifications on logout

- Clear notifications from localStorage

```typescript
logout(hostname?: string): void {
  this.logoutSubject.next();

  if (hostname) {
    this.clearTokenByHost(hostname);
    this.credentialsCache.delete(hostname);
    console.log(`Logged out from ${hostname}`);
  } else {
    this.clearAllTokens();
    this.credentialsCache.clear();
    this._currentHostname = null;
    console.log('Logged out from all hosts');
  }

  // Clear notifications from localStorage
  localStorage.removeItem('notifications');
}
```

### 7. Update `addNotification` to not show toasts for old notifications

In `src/contexts/NotificationContext.tsx`, modify `addNotification` to:
- Check if the notification has already been shown as a toast
- If so, don't show it again

```typescript
const addNotification = useCallback(
  (notification: Notification) => {
    dispatch({ type: 'ADD_NOTIFICATION', notification });
    if (notification.showAsToast && !notification.alreadyShownToast) {
      const toastActions = notification.actions?.map((action) => ({
        ...action,
        onClick: () => {
          markAsRead(notification.id, notification.channel);
          action.onClick?.();
        },
      }));

      toastService.showToast({
        id: notification.id,
        message: notification.message,
        details: notification.details,
        type: notification.type as 'info' | 'success' | 'warning' | 'error',
        autoClose: notification.autoClose,
        autoCloseDuration: notification.autoCloseDuration,
        dismissible: notification.dismissible,
        actions: toastActions,
      });
    }
  },
  [markAsRead],
);
```

## Risks/Considerations

- **Performance**: The `loadStateFromStorage` function will need to check the user ID, but this is a simple string comparison
- **Data loss**: When a user logs out, their notifications will be cleared from localStorage. This is the desired behavior.
- **Backward compatibility**: Existing notifications in localStorage without a user ID will be treated as belonging to the current user. This is a reasonable default.
